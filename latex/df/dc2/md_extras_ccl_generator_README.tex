This little Python tool is designed to generate example data for connected component analysis and labeling problems. While the data generated from this tool is not meaningful from a physics perspective, it exposes the most important parameters from a computational performance point of view.

The configurable parameters are as follows\+:


\begin{DoxyItemize}
\item Number of detector modules
\item Size of each detector module (assumed to be square)
\item Mean hits per module
\item Variance of hits per module
\item Mean cells per hit
\item Variance of cells per hit
\end{DoxyItemize}\hypertarget{md_extras_ccl_generator_README_autotoc_md12}{}\doxysection{Generating test files}\label{md_extras_ccl_generator_README_autotoc_md12}
An example run of this program looks like this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\$ python3 ccl\_generator.py -\/N 1000 -\/S 5000 -\/-\/Mm 1.65 -\/-\/Ms 0.95 -\/-\/Hm 1.8 -\/-\/Hs 0.89 -\/C 100 -\/o my\_run}
\end{DoxyCode}


This will generate one hundred files, in the format {\ttfamily my\+\_\+run\+\_\+0000000000.\+csv} and so on. The files should in a C\+SV format readably by traccc.
